## References

### Requirements Based on Distributed Systems Best Practices

**Deterministic Scheduling:**
- [Kubernetes Deterministic Scheduling](https://kubernetes.io/blog/2023/08/15/pkgs-k8s-io-introduction/) - K8s approach to reproducible scheduling
- [Mesos Resource Allocation](https://mesos.apache.org/documentation/latest/architecture/) - Two-level scheduling with deterministic offers

**Admission Control:**
- [Kubernetes Admission Controllers](https://kubernetes.io/docs/reference/access-authn-authz/admission-controllers/) - Production-grade admission control patterns
- [Borg: Admission Control at Scale](https://research.google/pubs/pub43438/) - Google's approach to cluster admission control

**Backpressure:**

- [Netflix: Backpressure and Flow Control](https://netflixtechblog.com/netflix-edge-load-balancing-695308b5548c) - Real-world application

**Failure Semantics:**
- [Google SRE Book: Handling Failures](https://sre.google/sre-book/addressing-cascading-failures/) - Cascading failure prevention
- [AWS: Exponential Backoff and Jitter](https://aws.amazon.com/blogs/architecture/exponential-backoff-and-jitter/) - Clear failure handling patterns
- [Distributed Systems Theory for Practical Engineers](https://www.the-paper-trail.org/post/2014-08-09-distributed-systems-theory-for-the-distributed-systems-engineer/) - Foundational concepts

**General Distributed Systems:**
- [Designing Data-Intensive Applications](https://dataintensive.net/) - Martin Kleppmann's authoritative text
- [MIT 6.824: Distributed Systems](https://pdos.csail.mit.edu/6.824/) - Academic foundation with practical labs
